# Event Storming Workshop вҖ” TicketHub

> Bir oda, bir duvar, renkli post-it'ler, ve 3-5 kiЕҹilik bir ekip.
> Amac: Sistemi kod yazmadan ONCE kesiflestirmek.

---

## Event Storming Nedir?

Alberto Brandolini'nin icad ettigi bir workshop teknigi.
Kurallar basit:

1. Buyuk bir duvara kaДҹДұt ser
2. Renkli post-it'lerle is akisini KRONOLOJIK olarak modelle
3. Herkes katilinir вҖ” developer, product owner, domain expert
4. Once "ne OLDU" (event) yaz, sonra "neden oldu" (command, actor, policy) ekle

---

## ADIM 6: Hot Spot вҶ’ Teknik Karar Tablosu

Workshop'ta cikan kirmizi post-it'ler (hot spot) вҶ’ teknik cozumler:

| Hot Spot (KIRMIZI post-it) | Tartisma | Karar | Koda yansimasi |
|---|---|---|---|
| "Ayni koltuДҹa 500 kisi ayni anda tiklarsa?" | DB lock mu? Application lock mu? | Redis distributed lock (Redlock) | `acquireSeatLock()` |
| "10dk doldu ama odeme tam o anda geldi?" | Race condition: expire vs confirm | Optimistic concurrency (version field) + expire() sessiz donme | `reservation.expire()` вҶ’ if not PENDING return |
| "Ayni odeme 2 kez gelirse?" | Network timeout вҶ’ retry вҶ’ cift odeme | Idempotency-Key header + Redis cache | `idempotencyGuard` middleware |
| "Stripe cokerse?" | Her istek 30sn timeout вҶ’ sunucu coker | Circuit breaker (5 hata вҶ’ open) | `CircuitBreaker` class |
| "Fiyat neden degisti?" | Musteri sikayet edecek, regГјlasyon | Append-only event log (event sourcing lite) | `pricing_events` tablosu |
| "Sahte QR bilet?" | Fotokopi, screenshot, manipule | HMAC-SHA256 imza + constant-time verify | `generateQRPayload()` |
| "Karaborsaci 100 bilet alirsa?" | Scalping вҶ’ gercek musteriler bilet bulamaz | Kullanici basina max 6 bilet | `MAX_SEATS_PER_BOOKING = 6` |
| "Etkinlige 2 saat kala iptal?" | Organizator zarar eder | 48 saat kurali | `CANCELLATION_DEADLINE_HOURS = 48` |

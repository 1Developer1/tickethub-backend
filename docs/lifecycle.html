Tam Proje Akisi — Bastan Sona

Bir biletin yolculugu: kullanici kayit → koltuk sec → ode → bilet al → kapida gir. Her adimda hangi moduldeki hangi state degisiyor.
1. Kullanici Kayit Olur
User.role = USER
argon2id hash + JWT access (15dk) + refresh (7gun) token uretilir
2. Organizator Etkinlik Olusturur
Event.status = DRAFT
Mekan secilir, tarih belirlenir, koltuk fiyatlari set edilir
3. Etkinlik Yayinlanir
Event.status = PUBLISHED
Arama sonuclarinda gorunur, bilet satisa acilir
4. Musteri Koltuk Secer
Reservation.status = PENDING SeatHold.status = HELD
Redis distributed lock → DB kontrol → entity olustur → DB kaydet
⚡ Policy: 10dk sonra expire BullMQ job'u kuruldu
5. Musteri Odeme Yapar
Payment.status = PENDING Payment.status = SUCCEEDED
Idempotency-Key kontrol → circuit breaker → Stripe adapter → DB kaydet
6. Rezervasyon Onaylanir
Reservation.status = CONFIRMED SeatHold.status = CONFIRMED
Entity guard: PENDING mi? Hold expired mi? → state transition → version++
⚡ Policy: QR uret + email gonder + fiyat recalculate (3 async job)
7. QR Bilet Uretilir (arka plan)
Ticket.status = VALID Notification.status = SENT
HMAC-SHA256 imzali QR kod + onay emaili (BullMQ worker)
8. Etkinlik Gunu — Kapida QR Okutulur
Ticket.status = USED
HMAC dogrula → DB kontrol → ilk kullanim → USED yap → "Hosgeldiniz!"
9. Etkinlik Tamamlanir
Event.status = COMPLETED
Etkinlik bitti — artik bilet satilmaz, yeni rezervasyon yapilamaz

Alternatif Yollar (mutsuz senaryolar)

4b. 10 Dakika Doldu — Odeme Yapilmadi
Reservation.status = EXPIRED SeatHold.status = RELEASED
BullMQ delayed job → entity.expire() → koltuklar serbest → baskasi alabilir
⚡ expire() sessiz donus — zaten CONFIRMED ise hicbir sey yapmaz (race condition guvenli)
6b. Musteri Iptal Etti (48+ saat once)
Reservation.status = CANCELLED SeatHold.status = RELEASED Payment.status = REFUNDED
entity.cancel() → 48h guard → koltuk serbest → odeme iade
5b. Odeme Basarisiz
Payment.status = FAILED
Reservation hala PENDING — kullanici tekrar deneyebilir (hold devam ediyor)
7b. Email Gonderilemedi
Notification.status = FAILED
3 retry sonra dead letter queue → bilet GECERLI, email sorunu ayri

Projenin kalbi. Entity command metotlari ile korunuyor — disaridan status degistirilemez.

Entity Korumali reservation.entity.ts

guard: status===PENDING && !expired
guard: hours > 48
Basarili gecis (confirmPayment)
Otomatik (expire — BullMQ)
Kullanici tetikli (cancel)
✗ Yasak Gecisler (entity THROW eder)
EXPIRED → CONFIRMED ✗ "suresi dolmus, onaylanamaz"
CANCELLED → CONFIRMED ✗ "iptal edilmis, onaylanamaz"
CONFIRMED → EXPIRED ✗ "onaylanmis, expire olmaz"
EXPIRED → CANCELLED ✗ "suresi dolmus, iptal anlamsiz"

Modul Etkilesimleri — Bir State Degisince Digerleri Nasil Etkileniyor

Bir modulun state'i degistiginde baska modullerin state'i de degisiyor. Zincir reaksiyon.

Reservation CONFIRMED olunca

Reservation: CONFIRMED→
ayni anda:
SeatHold: CONFIRMEDsync — ayni transaction
Ticket: VALIDasync — BullMQ
Notification: SENTasync — BullMQ
Pricing: recalculateasync — talep degisti

Reservation EXPIRED olunca

Reservation: EXPIRED→
SeatHold: RELEASEDsync — koltuk serbest
Pricing: recalculateasync — talep dustu

Reservation CANCELLED olunca

Reservation: CANCELLED→
SeatHold: RELEASEDsync — koltuk serbest
Payment: REFUNDEDasync — iade (eger CONFIRMED'dan geldiyse)
Ticket: REVOKEDsync — bilet iptal

Reservation PENDING olunca (olusturuldu)

Reservation: PENDING→
SeatHold: HELDsync — ayni transaction
BullMQ: expire jobgelecek — 10dk delay
Pricing: recalculateasync — talep artti
Zincir Reaksiyon Ozeti
Booking merkez — tum zincir buradan baslar. Booking'in state'i degistiginde:
  → SeatHold her zaman sync degisir (ayni transaction, tutarlilik garanti)
  → Payment sadece CANCELLED'dan async tetiklenir (iade)
  → Ticket sadece CONFIRMED'dan async olusur (QR uret)
  → Notification CONFIRMED/CANCELLED/EXPIRED'dan async tetiklenir (email)
  → Pricing her state degisikliginde async recalculate (talep degisti)

sync = ayni istek icinde, basarisiz olursa tumu geri alinir
async = BullMQ ile arka planda, basarisiz olursa retry, ana islem etkilenmez
